Class gears.history

A generic and stateful history tracking class.

This module hosts an abstract implementation of history tracking. It can be used with any datatype and hide the implementation details.

Here is the client focus history is implemented:

local h = gears.history {}
capi.client.connect_signal("focus"   , h.push_front)
capi.client.connect_signal("manage"  , h.push_back )
capi.client.connect_signal("unmanage", h.remove    )

Object properties

mode The history updating mode.
maximum_entries The maximum number of entries.
track_count Count the number of time an entry is used.
is_paused Is the history tracking currently paused.

Methods

gears.history:resume () Resume tracking.
gears.history:pause () Pause tracking.
gears.history:remove (value) Remove a value.
gears.history:push_front () Push an entry to the front (most recent) spot.
gears.history:push_back () Push an entry to the back (oldest) spot.
gears.history:pop_front () Pop (remove) the most recent entry.


Object properties

mode

The history updating mode.

Possible values are:

  • heap: Always keep a single instance of each entry (default)
  • multi: Allow an entry to be in multiple positions

Type:

  • string
maximum_entries
The maximum number of entries.

Type:

  • number
track_count
Count the number of time an entry is used.

Type:

  • boolean (default false)
is_paused
Is the history tracking currently paused.

When paused, all push will be ignored.

Type:

  • boolean
