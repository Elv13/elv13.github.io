Class awful.spawn.factory

A declarative client builder object.

This module offers an higher level API compared to awful.spawn. It is aimed at use cases where a full and unbroken pipeline between building a future client, spawning the process and using the client is required. Example use cases include layout serialization, advanced application launchers and database to remember and restore runtime properties the next time a client is spawned with the same command.

awful.spawn.factory:emit_signal (name, ...)
Emit a signal.
  • name string The name of the signal.
  • ... Extra arguments for the callback functions. Each connected function receives the object as first argument and then any extra arguments that are given to emit_signal().
awful.spawn.factory:connect_signal (name, func)
Connect to a signal.
  • name string The name of the signal.
  • func function The callback to call when the signal is emitted.
awful.spawn.factory:weak_connect_signal (name, func)
Connect to a signal weakly.

This allows the callback function to be garbage collected and automatically disconnects the signal when that happens.

Warning: Only use this function if you really, really, really know what you are doing.

  • name string The name of the signal.
  • func function The callback to call when the signal is emitted.
